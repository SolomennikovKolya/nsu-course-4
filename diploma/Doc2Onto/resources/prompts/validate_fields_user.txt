Тебе дан текст документа и результаты извлечения полей.

Твоя задача — провалидировать и при необходимости исправить значения полей.

### Входные данные

#### 1. Текст документа в формате UDDM
$document_uddm_text

#### 2. Результаты извлечения и предварительной валидации
$fields

### Требования

Для каждого поля:
1. Проверь соответствие значения описанию
2. Проверь наличие подтверждения в тексте документа
3. При необходимости исправь значение
4. Если восстановление невозможно — укажи причину

### Формат ответа

Формат ответа — строго JSON.
Никакого дополнительного текста.

Возможные статусы поля:
- "valid"      - значение корректно
- "corrected"  - значение было неверным, но сейчас исправлено
- "invalid"    - значение невозможно восстановить

Формат JSON-а:
{
  "field_name": {
    "status": "valid | corrected | invalid",  - статус поля
    "corrected_value": "строка или null",     - исправленное значение
    "error": "null или сообщение об ошибке"   - текст ошибки
  }
}

Возможные случаи:
- Если поле корректно → status = "valid", corrected_value = null, error = null
- Если исправлено → status = "corrected", corrected_value = исправленное значение, error = null
- Если невозможно восстановить → status = "invalid", corrected_value = null, error = сообщение ошибки (кратко и по делу)
