Сгенерируй описание шаблона извлечения знаний на основе приведённого документа и онтологии.

### Входные данные

#### 1. Название шаблона
$template_name

#### 2. Пример документа в формате UDDM
$example_uddm_text

#### 3. Онтологическая схема
$ontology_schema

### Требования к ответу

Ответ строго состоит из двух частей:

1. Краткая справка о шаблоне (1–3 предложения). Кратко: что это за документ, какие сущности он содержит, какие классы / свойства онтологии затрагивает. Без вступительных слов «краткая справка», «описание» и т. п.

2. Список извлекаемых полей в виде Markdown-таблицы:

| Название поля | Описание | Где искать | Как извлекать | Маппинг в онтологию |
|---------------|----------|------------|---------------|----------------------|

Где:
- **Название поля** — snake_case, уникальное.
- **Описание** — краткое смысловое описание (для человека и LLM).
- **Где искать**: тип блока (текст / параграф / список / таблица / ячейка), якорный текст («ФИО», «Студент», «Оценка»), относительное положение («параграф после X», «ячейка справа от Y», «строка под заголовком Z», «следующий параграф после маркера»).
- **Как извлекать**: формат значения и способ извлечения (например: «ФИО, 2–3 слова кириллицей, формат „Фамилия Имя [Отчество]“», «дата формата DD.MM.YYYY», «целое число от 1 до 4», «текст после двоеточия», «regex `(\\d{2}\\.\\d{2}\\.\\d{2})`»).
- **Маппинг в онтологию** — на какой класс / свойство онтологии это поле отобразится при построении триплетов. Указывай в виде `:Класс / :свойство` (например: `:Студент / :фамилия+:имя+:отчество`, `:Практика / :датаНачалаПрактики`, `:ВКР / :темаВКР`, `:Сотрудник / :занимаетДолжность → :Должность_Доцент`). Если для поля используется хеш-IRI — пометь это явно (например: «хеш :Персона по ФИО»).

Дополнительно:
- Извлекай только варьирующиеся поля; константы, шапки, фиксированные подписи — пропускай.
- Поля должны быть атомарными (например, не объединяй ФИО+должность в одном поле — это два разных поля).
- Не дублируй поля и не оставляй пустых строк в таблице.
- Если документ не содержит данных, отображаемых на онтологию, верни справку и пустую таблицу с заголовком (только шапка таблицы).
- Если поле не мапится на онтологию, НЕ добавляй его в таблицу.

### Пример ожидаемого ответа

Заявление студента о выборе темы ВКР: фиксирует автора-студента, тему работы, руководителя ВКР и его должность. Пополняет :Студент, :ВКР и :Сотрудник.

| Название поля | Описание | Где искать | Как извлекать | Маппинг в онтологию |
|---------------|----------|------------|---------------|----------------------|
| student_name | ФИО обучающегося | Параграф со словом «(Ф.И.О.)» | Текст до «(Ф.И.О.)», ФИО кириллицей, 2–3 слова | хеш :Студент по ФИО + :фамилия / :имя / :отчество / :фио |
| group_number | Номер учебной группы | Параграф с «группы №» | regex `группы\s*№\s*(\d+)`, целое число | :Группа (локальный IRI :Группа_<номер>) + :номерГруппы; связь :Студент :обучаетсяВГруппе |
| thesis_title | Тема ВКР | Параграф после маркера «Тема» | Текст между кавычками | хеш :ВКР по теме, :темаВКР; :Студент :имеетВКР |
| supervisor_name | ФИО руководителя ВКР | Параграф с «Руководитель ВКР» | regex с тремя словами кириллицы | хеш :Сотрудник по ФИО + :фамилия / :имя / :отчество / :фио; связь :ВКР :руководительВКР |
| supervisor_position | Должность руководителя ВКР | Параграф с «Руководитель ВКР» | regex по списку должностей («доцент», «зав. кафедрой», …) | :Сотрудник :занимаетДолжность → ONTO.Должность_<Значение> |
