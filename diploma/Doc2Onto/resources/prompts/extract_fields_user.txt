Тебе дан документ и список всех полей с результатом декларативного (шаблонного) извлечения.

Твоя задача — для каждого поля:
- если template-значение присутствует, оцени его корректность относительно description
  - если корректно — подтверди
  - если значение некорректно — попробуй найти корректное значение в документе
- если template-значение отсутствует — попробуй извлечь значение из документа

### Входные данные

#### 1. Документ в формате UDDM
$document_uddm

#### 2. Поля (JSON)
$fields

### Требования
- Используй описание поля как подсказку (что именно искать)
- Учитывай возможные вариации формулировок
- Если значение не найдено — верни null
- Не извлекай лишний текст (только значение)

### Формат ответа

Формат ответа:
- Возвращай СТРОГО JSON-объект, где ключи — имена полей.
- Значение по каждому полю — объект:
  - status: boolean (true если итоговое значение корректно)
  - value: string или null (итоговое значение)
  - error: string или null (ошибка/предупреждение)

Формат ответа — строго JSON вида:
{
  "field_name": {
    "status": true | false,
    "value": "строка" | null,
    "error": "строка" | null
  },
  ...
}

Пояснение к формату:
- field_name: название поля
- status: true если итоговое значение корректно
- value: итоговое значение
- error: ошибка/предупреждение (всегда кратко и по делу)

Логика работы (по сценариям):
- шаблон изначально извлёк поле корректно: status = true, value = null, error = null
- шаблон извлёк некорректное поле, но тебе получилось извлечь правильное значение: status = true, value = <корректное значение>, error = <предупреждение, что было не так>
- шаблон извлёк некорректное поле, и тебе тоже не удалось его извлечь: status = false, value = null, error = <ошибка>
- шаблон вообще не смог извлечь поле, но тебе получилось его извлечь: status = true, value = <значение поля>, error = null
- шаблон вообще не смог извлечь поле, и тебе тоже не получилось его извлечь: status = false, value = null, error = <ошибка>
