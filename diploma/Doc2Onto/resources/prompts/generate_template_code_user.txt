Сгенерируй Python-код шаблона извлечения по описанию ниже.

### Входные данные

#### 1. Описание шаблона
$template_description

#### 2. Пример документа в формате UDDM
$example_uddm_text

#### 3. Каркас, который надо заполнить
$code_example

### Требования

Реализуй класс TemplateCode (методы classify, fields и build).

#### classify(self, doc_name: str, uddm: UDDM) -> bool
Двухуровневая классификация:
1. Сначала по имени документа (`doc_name.lower()`): несколько ключевых слов через `all(word in name for word in [...])`. Достаточно специфично, чтобы не путать с однотипными классами (например, «отзыв» — недостаточно специфично для «Отзыв руководителя практики Бакалавриат ПИиКН 7 сем»). Минимум — должно срабатывать для прилагаемого примера документа.
2. Затем фолбэк по содержимому UDDM: `str(uddm.root).lower()` + проверка набора характерных якорных фраз.

#### fields(self) -> List[Field]
Каждый Field соответствует одному полю из «Описания шаблона».
- `name` — snake_case, уникальное в шаблоне.
- `description` — берётся из описания (короткое осмысленное).
- `selector` — строго по разделу «Где искать». Предпочитай относительные позиции (`next_element()` / `previous_element()` / `inner_element(...)` / `outer_element()`) — структура UDDM может быть зашумлена.
- `extractor` — строго по разделу «Как извлекать». Помни, что в выбранном строковом представлении кроме нужного значения часто есть мусор (заголовки полей, маркеры). Не делай trim в конце цепочки (он автоматический).
- `validator` — максимально полезные, но не избыточные проверки. Лучше пустой `val()`, чем бесполезный `min_length(3)`.
- При нехватке DSL — кастомные функции через `apply(...)` в селекторе/экстракторе/валидаторе.

#### build(self, b: TemplateGraphBuilder)
- Используй ТОЛЬКО классы и свойства из онтологической схемы (см. системный промпт). Никаких выдуманных предикатов.
- Для всех индивидов классов с подсказкой `IRI: b.<helper>(...)` ИСПОЛЬЗУЙ соответствующий хелпер билдера. Хелперы сами добавляют тип и идентифицирующие литералы — НЕ дублируй их вручную.
- Перечисления (Должность, УченаяСтепень, УченоеЗвание, ВидПрактики, Оценка) выбираются ТОЛЬКО через хелперы `b.position` / `b.degree` / `b.title` / `b.practice_kind` / `b.grade`. Запрещено заводить в шаблоне локальные таблицы синонимов вроде `_POSITION_MAP` или приватные функции для перечислений: эта логика уже в DSL.
- Для опциональных свойств (например, должность сотрудника, степень, звание) используй `b.add_object_property_optional(...)` — он молча пропустит триплет, если значение отсутствует или не распознано.
- Низкоуровневое API (`b.field(...).transform(...)`, лямбды) применяй ТОЛЬКО там, где нет подходящего хелпера и где трансформация специфична для данного шаблона.

### Дополнительно

- Если в описании шаблона нет чёткого списка полей — сам выбери разумный набор полей, ОПИРАЯСЬ на онтологию (только то, что варьируется от документа к документу в рамках одного класса).
- НЕ извлекай постоянную информацию (например, шапки, фиксированные заголовки), её не нужно класть в граф.

### Формат ответа

- Верни ПОЛНЫЙ исходник Python-файла шаблона, начиная с импортов.
- Без объяснений, без markdown, без тройных бэктиков, без лишнего текста.

Перед выводом проверь:
- все поля из описания присутствуют;
- нет несуществующих методов / namespace-ов / предикатов;
- классы и предикаты из `ONTO.*` действительно есть в схеме выше;
- для каждого индивида типа :Персона / :Организация / :Профиль / :Группа / :НаправлениеПодготовки / :ВКР / :Практика использован соответствующий хелпер `b.<helper>(...)`;
- для каждого индивида перечисления использован соответствующий хелпер (`b.position` / `b.degree` / `b.title` / `b.practice_kind` / `b.grade`);
- в шаблоне НЕТ модульных констант-таблиц синонимов и приватных функций для перечислений;
- код синтаксически корректен.

### Пример готового шаблона (для ориентира)

from typing import List

from core import *


class TemplateCode(BaseTemplateCode):
    def classify(self, doc_name: str, uddm: UDDM) -> bool:
        name = (doc_name or "").lower()
        if all(word in name for word in ["заявление", "на", "практику", "бакалавриат", "пиикн"]):
            return True

        full_text = str(uddm.root).lower()
        return all(word in full_text for word in ["заявление", "прошу направить меня", "практику"])

    def fields(self) -> List[Field]:
        return [
            Field(
                "student_name", "ФИО обучающегося",
                sel().find(ElementType.P, Predicate.contains_text("(Ф.И.О.)")),
                ext().before("(Ф.И.О.)").keep_letters_and_spaces().normalize_spaces(),
                val().alphabetic().word_count(min_words=2, max_words=3),
            ),
            Field(
                "group_number", "Номер учебной группы",
                sel().find(ElementType.P, Predicate.contains_text("группы №")),
                ext().regex(r"группы\s*№\s*([0-9]+)", group=1),
                val().integer(),
            ),
            Field(
                "direction_code", "Код направления подготовки",
                sel().find(ElementType.P, Predicate.contains_text("направление")),
                ext().regex(r"(\d{2}\.\d{2}\.\d{2})", group=1),
                val().regex(r"\d{2}\.\d{2}\.\d{2}", full_match=True),
            ),
            Field(
                "direction_name", "Наименование направления подготовки",
                sel().find(ElementType.P, Predicate.contains_text("(код и наименование направления)")).previous_element(),
                ext().regex(r"\d{2}\.\d{2}\.\d{2}\s*(.+)$", group=1),
                val(),
            ),
            Field(
                "profile_name", "Наименование профиля подготовки",
                sel().find(ElementType.P, Predicate.contains_text("направленность (профиль)")),
                ext().after("направленность (профиль)"),
                val(),
            ),
            Field(
                "organization_full_name", "Полное наименование организации места практики",
                sel().find(ElementType.P, Predicate.contains_text("(указывается наименование практики)")).previous_element(),
                ext(),
                val(),
            ),
            Field(
                "supervisor_name", "ФИО руководителя ВКР",
                sel().find(ElementType.P, Predicate.contains_text("Руководитель ВКР")),
                ext().regex(
                    r"Руководитель ВКР.*?([А-ЯЁ][а-яё-]+\s+[А-ЯЁ][а-яё-]+\s+[А-ЯЁ][а-яё-]+)",
                    group=1,
                ),
                val().alphabetic().word_count(exact=3),
            ),
            Field(
                "supervisor_position", "Должность руководителя ВКР",
                sel().find(ElementType.P, Predicate.contains_text("Руководитель ВКР")),
                ext().regex(
                    r"(зав\.\s*кафедрой|доцент|профессор|старший\s+преподаватель|преподаватель)",
                    group=1,
                ),
                val(),
            ),
        ]

    def build(self, b: TemplateGraphBuilder):
        student = b.person("student_name", role=ONTO.Студент)
        b.add_object_property(student, ONTO.обучаетсяВГруппе, b.group("group_number"))
        b.add_object_property(
            student, ONTO.обучаетсяНаНаправлении,
            b.direction("direction_code", name_field="direction_name"),
        )
        b.add_object_property(student, ONTO.имеетПрофиль, b.profile("profile_name"))

        b.organization("organization_full_name", role=ONTO.ВнешняяОрганизация)

        supervisor = b.person("supervisor_name", role=ONTO.Сотрудник)
        b.add_object_property_optional(
            supervisor, ONTO.занимаетДолжность, b.position("supervisor_position"),
        )

        thesis = b.thesis(student=student)
        b.add_object_property(thesis, ONTO.руководительВКР, supervisor)
        b.add_object_property(student, ONTO.имеетВКР, thesis)
