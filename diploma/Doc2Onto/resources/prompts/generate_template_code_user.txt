Сгенерируй Python-код шаблона извлечения.

### Входные данные

#### 1. Описание шаблона
$template_description

#### 2. Пример документа в формате UDDM
$example_uddm_text

#### 3. Код шаблона, который тебе надо взять за основу и заполнить
$code_example

### Требования

Реализуй класс TemplateCode (реализуй методы classify, fields и build_triples)

#### 1. Требования к classify
Используй двухуровневую классификацию:
1. Классификация по имени документа:
  - Использую ключевые слова
  - Классификация по имени не должна быть универсальной, но как минимум она должна работать для переданного тебе примера документа
  - Классификация должна быть специализированной (например, наличие слова "отзыв" не говорит о том, что документ имеет класс "Отзыв руководителя практики Бакалавриат ПИиКН 7 сем")
2. Классификация по UDDM:
  - Используй характерные текстовые якоря
  - Не делай сложную логику
  - Можешь использовать все методы прямого обрашения к структуре UDDM

#### 2. Требования к fields
Каждый Field должен соответствовать одному полю из описания.

Параметры Field:
- name - название в snake_case
- description - описание поля из описания шаблона 
- selector
  - Строго по "Где искать"
  - Стараться использовать относительные позиции элементов (так как структура UDDM может быть не идеальна и извлекаться с ошибками из исходного документа)
  - Если в DSL нет подходящих методов для селекции поля, использовать кастомные функции через apply
- extractor
  - Строго по "Как извлекать"
  - Не забывать, что в строке помимо искового значения поля могут быть и мусорные 
  - Не делать trim в конце цепочки (он и так делается автоматически в ходе работы пайплайна)
  - Если в DSL нет подходящих методов для извлечения поля, использовать кастомные функции через apply
- validator
  - Максимально строгие, но при этом логичные проверки (чтобы не отбросить на самом деле валидное значение поля)
  - Лучше оставить цепочку валидации пустой, чем писать бесполезные проверки по типу `min_length(3)`
  - Если в DSL нет подходящих методов для валидации поля, использовать кастомные функции через apply

#### 3. Требования к build_triples
- Используй все необходиные возможности rdflib
- Главное - построить граф RDF со всеми необходимыми знаниями
- Опирайся на переданную онтологию, чтобы согласовать ресурсы
- Используй FieldsAccessor для создания Literal или URIRef из полей шаблона
- Базовый namespace: @prefix : <http://doc2onto.org/ontology#> .

#### Примечание
В случае, если описание шаблона пустое или не содержит четкого описания извлекаемых полей, сам придумай, какие поля извлекать.
Только извлекай поля, значения которых варьируются от документа к документу в рамках одного класса (постоянную информацию извлекать не надо).

### Ответ

Формат ответа:
- Полный текст кода (включая импорты)
- Без пояснений
- Без markdown
- Без ``` блоков

Перед выводом кода проверь:
- все поля из описания присутствуют
- нет несуществующих методов
- код компилируется логически

### Примеры

#### Пример шаблона "Заявление на практику Бакалавриат ПИиКН 7 сем"

from typing import List, Dict

from core.template.base import BaseTemplateCode
from core.fields.field import Field
from core.fields.field_selector import *
from core.fields.field_extractor import *
from core.fields.field_validator import *
from core.uddm.model import *


class TemplateCode(BaseTemplateCode):

    def classify(self, doc_name: str, uddm: UDDM) -> bool:
        name = (doc_name or "").lower()
        if all(word in name for word in ["заявление", "на", "практику"]):
            return True

        full_text = str(uddm.root).lower()
        return all(word in full_text for word in ["заявление", "прошу направить меня", "практику"])

    def fields(self) -> List[Field]:
        return [
            Field(
                name="student_name",
                description="ФИО обучающегося",
                selector=select().find(ElementType.P, contains_text("(Ф.И.О.)")),
                extractor=extract().before("(Ф.И.О.)"),
                validator=validate().alphabetic().word_count(exact=3),
            ),
            Field(
                name="course_number",
                description="Номер курса обучения",
                selector=select().find(ElementType.P, contains_text("курса")),
                extractor=extract().regex(r"(\d+)\s*курса", group=1),
                validator=validate().integer().in_range(1, 4),
            ),
            Field(
                name="group_number",
                description="Номер учебной группы",
                selector=select().find(ElementType.P, contains_text("группы №")),
                extractor=extract().regex(r"группы\s*№\s*([0-9]+)", group=1),
                validator=validate().integer(),
            ),
            Field(
                name="direction_code",
                description="Код направления подготовки",
                selector=select().find(ElementType.P, contains_text("направление")),
                extractor=extract().regex(r"(\d{2}\.\d{2}\.\d{2})", group=1),
                validator=validate().regex(r"\d{2}\.\d{2}\.\d{2}", full_match=True),
            ),
            Field(
                name="direction_name",
                description="Наименование направления подготовки",
                selector=select().find(ElementType.P, contains_text("(код и наименование направления)")).previous_element(),
                extractor=extract().after("направление ").after(" ").trim(),
                validator=validate().alphabetic(),
            ),
            Field(
                name="profile_name",
                description="Наименование профиля подготовки",
                selector=select().find(ElementType.P, contains_text("направленность (профиль)")),
                extractor=extract().after("направленность (профиль)").trim(),
                validator=validate().alphabetic(),
            ),
            Field(
                name="organization_full_name",
                description="Полное наименование организации места практики",
                selector=select().find(ElementType.P, contains_text("(полное название организации")).previous_element(),
                extractor=extract(),
                validator=validate(),
            ),
            Field(
                name="application_date",
                description="Дата заявления",
                selector=select().find(ElementType.P, contains_text("Дата:")),
                extractor=extract().after("Дата: ").regex(r"«?\s*(\d{1,2})\s*»?\s*([А-Яа-яЁё]+)\s*(\d{4}\s*г\.)", group=0),
                validator=validate(),
            ),
            Field(
                name="supervisor_name",
                description="ФИО руководителя ВКР",
                selector=select().find(ElementType.P, contains_text("Руководитель ВКР")),
                extractor=extract().regex(r"Руководитель ВКР.*?(?:\(?Ф\.И\.О\. полностью\)?\s*)?(.*?)(?:\s+зав\. кафедрой|\s+\(должность\)|$)",
                                          group=1).keep_letters_and_spaces().normalize_spaces(),
                validator=validate().alphabetic().word_count(exact=3),
            ),
            Field(
                name="supervisor_position",
                description="Должность руководителя ВКР",
                selector=select().find(ElementType.P, contains_text("Руководитель ВКР")),
                extractor=extract().regex(r"Руководитель ВКР.*?(?P<position>зав\.\s*кафедрой|доцент|профессор|старший\s+преподаватель|преподаватель)", group="position"),
                validator=validate(),
            ),
        ]

    def build_triples(self, fields_values: Dict[str, str]) -> List[Dict]:
