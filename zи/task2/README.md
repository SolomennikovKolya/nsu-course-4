
## Сканер безопасности

### Задание

В рамках лабораторной будет изучен функционал сканера безопасности.

ScanOVAL предназначена для оперативного автоматизированного обнаружения уязвимостей программного обеспечения на рабочих станциях и серверах, функционирующих под управлением операционных систем семейства Microsoft Windows.

Задачи на лабораторную работу:
1. Установить сканер безопасности на ПК или развернуть виртуальный образ. Это может быть, как Ваш личный ПК, так и образ Win7, который можно скачать по ссылке (https://disk.yandex.ru/d/0yB2PV0UvPaS9g).
2. Загрузить в сканер ScanOVAL базу данных уязвимостей (согласно инструкции).
3. Произвести сканирование ПК и выгрузить отчет о найденных уязвимостях.
4. Найти описание путей устранения трех наиболее критичных уязвимостей и приложить к отчету.

Дистрибутив сканера, база уязвимостей, мануал по сканеру и образ Win7 находятся в папках (https://disk.yandex.ru/d/0yB2PV0UvPaS9g). Пароль от учетной записи на Win7 – Password123.

В результате от Вас жду отчет о сканирование и описание путей устранения трех уязвимостей.

#### Некоторые критические уязвимости

**1. BDU:2016-01135 Уязвимость в Oracle Java до 6.0.114 и до 7.0.100 и до 8.0.78**

Описание уязвимости: Неопределённая уязвимость в Oracle Java SE 6u113, 7u99, и 8u77 позволяет удалённым злоумышленникам повлиять на конфиденциальность, целостность и доступность через векторы, связанные с 2D.

Решение: Использование рекомендаций производителя: http://www.oracle.com/technetwork/security-advisory/cpuapr2016v3-2985753.html.

Там пишут: В связи с этим Oracle настоятельно рекомендует заказчикам использовать активно поддерживаемые версии и без промедления применять критические исправления в обновлениях. То есть просто надо обновить версию jdk.

**2. BDU:2024-07084 Уязвимость в Яндекс.Браузер до 24.7.1 (YandexBrowser)**

Описание уязвимости: Ошибка реализации в V8 в Яндекс.Браузер позволяет удалённым злоумышленникам вызвать повреждение кучи через специально сформированную HTML страницу.

Решение: Использование рекомендаций: Для программных продуктов Google Inc.: https://chromereleases.googleblog.com/2024 Для Debian GNU/Linux: https://security-tracker.debian.org/tracker/CVE-2024-7965 Для программных продуктов Novell Inc.: https://www.suse.com/sec.

Там пишут: Обновление программного обеспечения chromium до версии 131.0.6778.264+repack-1~deb12u1.osnova2u1.

Также описаны компенсирующие меры:
- использование средств антивирусной защиты с функцией контроля доступа к веб-ресурсам;
- контролируемый доступ в сеть Интернет – регламентация разрешенных сетевых ресурсов и соединений;
- запуск веб-браузера от имени пользователя с минимальными возможными привилегиями в операционной системе;
- применение систем обнаружения и предотвращения вторжений.

**3. BDU:2025-09829 Уязвимость в PostgreSQL начиная с 17.0 до 17.6 и начиная с 16.0 до 16.10 и начиная с 15.0 до 15.14 и начиная с 14.0 до 14.19 и начиная с 13.0 до 13.22 (3118)**

Описание уязвимости: Уязвимость в PostgreSQL до 17.6 и до 16.10 и до 15.14 и до 14.19 и до 13.22 позволяет выполнить произвольный код в psql-клиенте и на целевом сервере.

Решение: Использование рекомендаций: Для PostgreSQL: https://www.postgresql.org/support/security/CVE-2025-8714/ Для Postgres Pro Certified: https://postgrespro.ru/products/postgrespro/certified

Опять же, для решения надо просто обновить ПО до версии, где данная уязвимость исправлена.
